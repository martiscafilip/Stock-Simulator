1.Introduction

  • 1.1 Purpose

    In this document we will present the "stock-simulator" project v1.0. This project aims to help investment beginners to become familiar with the system without losing equity.

  • 1.2 Document Conventions

    This project will use an API to retrieve stock price data and a DB to store information about each user.

  • 1.3 Intended Audience and Reading Suggestions

    "Stock-Simulator" is useful for all people who want to trade on the stock market but also for people who are already familiar with the stock market, to strengthen their investment strategies.

  • 1.4 Product Scope

    First of all, the purpose of the project is to give users an easy-to-use interface with useful statistics and information. One benefit is to remove the influence of emotions by using virtual money, and the goal is to familiarize as many people with the stock market, becauze money is an important part of our lives that many people neglect.

3.Relevant code

  • 3.1.Leaderboard

    Method `updateList` takes as parameter a list from DB in json format. He delets the entire div with the leaderboard and create it again step by step. It is called in `.then` of a fetch

  • 3.2.Change Profile

    Method `createDemo` creates a json with needed informations for creating an account and make a `POST` request to "https://stock-simulator-hodler.herokuapp.com/app/api/createDemoAcc.php" to create it.

  • 3.3.Account

    Method `refreshAccount` is called from 20s to 20s and call `fetchAccountInfos` and `updateSessionCurrency` in order to update the infromations about the account. `fetchAccountInfos` takes current informations from DB in json format and takes over to `updateSessionCurrency` in order to modify the asked values.

  • 3.4.Trade

    The trade page contains at the top a list of stocks / coins that you can buy / sell. The selected stock / cryptocurrency will update the TradingView Widget (https://www.tradingview.com) which will indicate its price in real time.

    When you place a purchase order, it will be validated if it can be executed, and will extract the current price of the share / cryptocurrency using the Finhub app (https://finnhub.io) to insert the transaction in the Postgres database from Heroku.

    When we want to sell a transaction, the profit will be calculated, it will be added to the current balance from the database and the transaction will be deleted from the database.
